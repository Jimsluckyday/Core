Capper leaderboard — internal, not public

Ranked purely by win % once a capper clears the minimum — total pick count never drives the ranking, it's only shown as context. Default minimum is 2 graded picks, since a single pick isn't a meaningful percentage. Results open in a new tab so they don't interrupt data entry. "All time" clears filters and runs immediately — the typical view.

Capper performance by bet type — internal, not public

Each row is one capper's record within a single bet type (e.g. "King Cap · Spread"), not their overall record — so a capper who's mediocre overall can still show up strong on one bet type. Sorted by win % within each qualifying row, same minimum-picks logic as the leaderboard. Use this to decide which of a capper's bet types are actually worth publishing, rather than promoting or hiding a capper wholesale.

Consensus picks — internal, not public yet

A consensus requires the exact same selection AND line (where applicable) — "Spurs -6" and "Spurs -6.5" are counted separately, on purpose. Uncheck the box above to see every qualifying consensus for deeper internal review — the default shows just the strongest one per day, which is the version worth showing end users later.

Publish ranking — internal, not public

Scores every currently-pending pick using four equally-weighted signals (25% each): win % in this exact sport + bet type combo (e.g. Tennis Spreads specifically, not Spreads in general), overall win %, current streak (a bad streak actively drags the score down, not just withholds a bonus), and confluence — a bonus or PENALTY when other cappers independently picked this exact same game/selection/line. An agreeing capper above a 50% record helps the score; one below 50% actively hurts it, and the effect grows stronger both with a larger losing sample and with more agreeing cappers (capped at 3+). The sport+bet-type rate blends toward that capper's broader all-sport rate for that bet type until enough sport-specific picks pile up — so "2-0 in Tennis Spreads" leans on their overall Spread record at first, rather than either treating 2-0 as proven or ignoring it as meaningless. All win percentages are otherwise sample-size-adjusted the same way — a lucky small sample can't out-rank a proven track record. Click the small "N agreeing ▾" note under any score to see exactly which other cappers contributed to that pick's confluence bonus. Sort by Score to find the best candidates, then click Publish on however many you want featured.

Top & bottom cappers — internal, not public yet

Top 5 and Bottom 5 shown as separate lists, ranked purely by win %. Default minimum is 2 graded picks — a single pick is too small a sample to mean anything (100% off one pick isn't a real record), but 2+ gets ranked on percentage alone. Raise the minimum if you want a stricter bar. Value column estimates $100-per-pick profit using whichever odds are on file (opening if present, otherwise closing) — picks with no odds logged are excluded from that total, so it's a partial number until odds entry is more complete.

Capper streaks — internal, not public yet

Shows every capper currently on a win or loss streak at or above the minimum length — separate from win % or ranking, so a capper can show up here even if their overall record looks fine. Streak counts consecutive picks (a push doesn't break it), most recent first.

Capper analysis — internal, not public yet

"Total" counts every pick as its own data point. "Unique" collapses picks that are the exact same bet on the exact same game (same date, sport, bet type, selection, and line) down to one — so if five cappers all picked "Chiefs -6," that's 5 total picks but 1 unique bet. Also broken down by sport and by bet type, so you can see where accuracy is actually strongest, not just overall.

Duplicate pick checker — internal, not public

Finds picks that are an exact match on capper, date, sport, bet type, selection, line, and (for props) player/stat — the kind of thing that happens when the same file gets bulk-imported twice, or two people enter the same capper's picks without realizing it. Review each group and delete the extra copy — the original stays untouched, this never deletes automatically.

Bulk import from spreadsheet

Paste rows copied from a spreadsheet (or built here in chat) — first row must be column headers. Required columns: date, capper, sport, bet type, selection. Optional: line, home_away, odds, prop_player, prop_stat (or just "stat"), confidence, notes, published. For player props, if you don't include a separate prop_player column, the player's name is automatically taken from selection. Capper/sport/bet type must already exist in this tool (added via the Cappers/Sports/Bet Types section above) — unmatched names get flagged instead of guessed at.
